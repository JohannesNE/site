Reproducible reports with R Markdown

By Luke W. Johnston, Maria Izabel Cavassim Alves (contributed), Sara Tomiolo | December 13, 2019

install.packages("rmarkdown")

Objectives

  1. To become aware of the importance of reproducibility of data analyses.
  2. To learn how to generate high quality reports that can be shared with a broader audience.
  3. To create reproducible documents interwoven with R code that can be run on updated or different data sets.
  4. To use Rstudio not as a data science tool but also as a text editor and compiler!
  5. To know where to go for continued learning.

At the end of this session you will be able:

  • Create a new RMarkdown document.
  • Write text, headers, citations, and other report writing tasks in Markdown.
  • Insert R code chunks that will create figures, tables, or numbers.
  • Generate an analytically reproducible Word or HTML document.

Why try to be reproducible?

First of a couple of definitions to avoid confusion:

Replicability: Repeating a study by performing independently an identical study.

Reproducibility: Generating the same exact results when using the same data and code. This means that if you gave someone your data, your code, and your paper, they should be able to generate the exact same numbers you reported.

Research shows that often this is the case… even when the researchers try to reproduce their own results! A recent survey indicated that 70% of researchers failed to reproduce another scientist’s results and more than 50% of researchers were not able to reproduce their own results.

Being reproducible and writing a reproducible document/manuscript does several things for you:

  1. Makes your science better.
  2. Makes you more efficient and productive, since less time is spent between coding and putting your results in the document, or in searching where you put your code, your graphs and your manuscript.
  3. Makes you more confident in your results, since what you report is what you actually got from the analysis.
  4. Its actually a lot of fun! ;)

What is R Markdown?

A markup syntax that allows you to write a plain text (e.g. R code and scripts) document that can convert to a range of document formats, such as Word, PDF, html.

In R Markdown it is possible to write text and integrate it with code. In R scripts you would need to use #so that your comments are not read as code.

In R and R Markdown, the software that allows the conversion from R code into different document formats is called pandoc.

Let’s get to learning how to use R Markdown!

YAML Header

Every R Markdown documents starts with a section at the top called YAML Header. In this section you want to indicate the desired output of your document. Today we will output our R Markdown documents in html format, but you can also choose PDF, and Word documents.

If you want to produce a PDF document, you will need to install tinytex.

If you want to convert to Word and set up some features of your document (suck as line spacing and styles) you should create a reference style document. For this you can follow this very detailed tutorial or watch this video

The best part! Inserting R code

One of the most powerful and useful features of Rmarkdown, is its ability to combine text and code in the same document! You can insert plots by including a code chunk like below. The options added to the code chunk tell it to add a caption, and set the height and width of the figure.

We do this by inserting a code chunk. A code chunk can be inserted pressing Ctrl + Alt + I or Cmd + Option + I for Mac users.

```{r plot_sepal, fig.cap="Figure title here", fig.height=8, fig.width=8, echo=FALSE}
plot(Sepal.Length ~ Sepal.Width, data = iris)
```

You can also hide the code chunk but keep the output by using the echo=FALSE option:

```{r, echo=FALSE}
cor(iris$Sepal.Length, iris$Sepal.Width)
```

In this case we want to keep the code chunk visible and we also want to add some comments regarding our graph.

You can also create tables by using the kable() function in the knitr package.

```{r}
knitr::kable(head(iris), caption = "Table caption here")
```

The knitr:: part of the code tells R where the kable function comes from. So the :: is telling R “hey, look in the knitr package for the kable function”.

For small and not too complex tables, you can also create it outside of a R chunk using the Markdown syntax:

|       &nbsp;       | diverged | polymorphic |
|:------------------:|:--------:|:-----------:|
|   **synonymous**   |   1300   |    2109     |
| **non-synonymous** |   700    |     891     |

gives…

  diverged polymorphic
synonymous 1300 2109
non-synonymous 700 891

Exercise: Create a plot and add a table to your R Markdown document

Time: 10 min

Open RStudio and create an R Markdown document:

File -> New File -> R Markdown Save the file and call the file exercise.Rmd

Create a chunck of code and create a plot using the database CO2.

  1. Use names(CO2) to find the names of columns from the CO2 dataset. Choose two columns. Create a code chunk with a caption fig.cap option and the fig.width of 10 and a fig.height of 6. Include in the code chunk the function, replacing the __ with the two columns you chose:

    ggplot(CO2, aes(x =___, y = ___)) +
  geom____()

  2. Create a table of the head() of the CO2 dataset using the R function knitr::kable().

  3. Add a few lines of text underneath your graph

Compile your document by pressing the icon Knit to HTML or by typing Ctrl-Shift-K.

Markdown syntax

Similarly to a text editor, R Markdown allows you to modify and customize the syntax.

Headers

Such as for a section, subsection, subsubsection, etc.

# Header 1

gives…

Header 1

## Header 2

gives…

Header 2

### Header 3

gives…

Header 3

Text formatting

  • **bold** gives bold.
  • *italics* gives italics.
  • super^script^ gives superscript.
  • sub~script~ gives subscript.

Lists

Unnumbered list:

- item 1
- item 2
- item 3

gives…

  • item 1
  • item 2
  • item 3

Numbered list:

1. item 1
2. item 2
3. item 3

gives…

  1. item 1
  2. item 2
  3. item 3

Block quotes

One can also create quotes:

> Block quote

gives…

Block quote

Exercise: Construct an outline for a (hypthetical) paper

Time: 10 min

In your R Markdown document, add the sections of your future manuscript. Include the following “Header 1” # sections:

  • Abstract
  • Introduction
  • Material and Methods
  • Results
  • Discussion

  • Conclusion

  • Write some random words below the Abstract section, while using bold and italics.
  • Include an unnumbered list below Introduction listing two or three fake objectives.

  • Include a “Header 2” (##) called “Statistical analysis” below “Material and Methods”.

Compile it by pressing the icon Knit to HTML or by typing Ctrl-Shift-K.

Adding footnotes, pictures, and links to your document

Footnotes can be done using the following command:

Footnote[^1]

[^1]: Footnote content

gives…

Footnote1

A .png .jpeg or .pdf image can be attached in the following way:

![image caption](img/logo-original.png)

gives…

image caption

And a link can be linked in the following format:

[Link](https://google.com)

Ending remarks

We hope that we’ve shown the power that comes with using R Markdown and that we’ve convinced you enough to try using R Markdown for writing your reports! Believe us, it can save soooo much time in the end after you’ve learned how to incorporate text with R code when writing any type of document that relies on results from data analysis or presentation. R Markdown is a powerful tool for reproducibility and for creating beautiful reports! There are many other amazing features of R Markdown that you can learn about from the resources above. Try it out and learn more about it!
